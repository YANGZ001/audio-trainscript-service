# Audio Bridge Service — Test Plan

## Manual Test Scenarios

### 1. Bilibili Audio Fallback Transcription (Golden Path)
- **Setup**: Select a Bilibili video known to have no subtitles (e.g. `https://www.bilibili.com/video/BV1VMR4BNEYq` but with subtitles disabled/deleted).
- **Execution**: 
  - Send `POST http://localhost:port/api/transcribe` with BVID.
- **Expected Results**:
  - SSE connection opens.
  - Receives progress stream (`downloading` $\rightarrow$ `uploading` $\rightarrow$ `transcribing`).
  - Finishes with a `done` event containing a valid timestamped JSON transcript.
  - A check of SQLite `transcripts` table confirms the record is saved.

### 2. Google Drive Shared File Download
- **Setup**: Upload a test `.m4a` file to Google Drive and share the folder with the Service Account email.
- **Execution**: 
  - Call `/api/transcribe` with `type: "gdrive"` and the file ID.
- **Expected Results**:
  - File is successfully resolved and downloaded to local temp disk.
  - Transcript generated by Gemini matches the speech content.

### 3. Direct Multipart File Upload (<100MB)
- **Setup**: Prepare an `.m4a` recording file (e.g., 25MB, ~15 mins).
- **Execution**:
  - Send a multipart request to `/api/transcribe` with the file attached.
- **Expected Results**:
  - Upload is accepted and streamed to local disk.
  - SSE feedback correctly tracks the stages.
  - Final output JSON matches.

## Automated Test Scenarios

- **Bilibili URL Parser Unit Tests**: Verify extraction of BVID and page parameters.
- **SQLite Database Integration Tests**:
  - Insert transcript $\rightarrow$ query transcript $\rightarrow$ verify exact content matches.
  - Set a mock record created_at to `datetime('now', '-3 years')` and run the TTL cron method. Verify that the record is successfully deleted, while a record created `now` remains untouched.
- **Gemini Parser Validation**: Verify that responses containing markdown-wrapped JSON (e.g. ```json ... ```) are cleanly parsed without throwing syntax errors.

---

## Cleanup and Resource Verification

- Verify that the local `temp/` folder is empty after every successful or failed request.
- Query Gemini File API list endpoint after a task run to ensure that the uploaded file has been deleted from Google's servers.
